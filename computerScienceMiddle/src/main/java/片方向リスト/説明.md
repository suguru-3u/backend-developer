# 片方向リスト（Singly Linked List）

## 概念

片方向リストは、**ノード**と呼ばれる要素が一方向につながったデータ構造です。

各ノードは以下の2つを持ちます：
- **data** : 格納する値
- **next** : 次のノードへの参照（最後のノードはnull）

---

## 図解

```
[data|next] → [data|next] → [data|next] → null
   head          2番目         末尾(tail)
```

具体的な例（1→2→3）：

```
[1|next] → [2|next] → [3|null]
  head
```

---

## 配列との比較

| | 片方向リスト | 配列 |
|---|---|---|
| 先頭への挿入 | O(1) ✅ | O(n) |
| 末尾への挿入 | O(n) | O(1) ✅ |
| 検索 | O(n) | O(1) ✅ |
| メモリ | 動的に確保 ✅ | 固定サイズ |
| 前の要素へのアクセス | ❌ 不可 | ✅ 可能 |

---

## 主な操作のコード例

**先頭に追加（最も得意な操作）**
```java
public void addFirst(E data) {
    SinglyLinkedListNode<E> newNode = new SinglyLinkedListNode<>(data);
    newNode.next = head;
    head = newNode;
}
```

**末尾に追加**
```java
public void addLast(E data) {
    SinglyLinkedListNode<E> newNode = new SinglyLinkedListNode<>(data);
    if (head == null) {
        head = newNode;
        return;
    }
    SinglyLinkedListNode<E> current = head;
    while (current.next != null) {
        current = current.next;
    }
    current.next = newNode;
}
```

**ノードの削除**
```java
public void remove(E data) {
    if (head == null) return;
    
    // 先頭が対象の場合
    if (head.data.equals(data)) {
        head = head.next;
        return;
    }
    
    SinglyLinkedListNode<E> current = head;
    while (current.next != null) {
        if (current.next.data.equals(data)) {
            current.next = current.next.next; // 対象ノードをスキップ
            return;
        }
        current = current.next;
    }
}
```

削除の図解：
```
削除前: [1] → [2] → [3] → null
                ↑削除したい

削除後: [1] → [3] → null
// [1]のnextを[2]ではなく[3]に向ける
```

---

## 双方向リストとの違い

片方向リストは前のノードに戻れないのが最大の制約です。それを解決したのが**双方向リスト**で、`prev`（前ノードへの参照）も持ちます。

```
片方向: [1] → [2] → [3] → null
双方向: [1] ⇄ [2] ⇄ [3]
```

---

片方向リストは**スタック**や**キュー**の実装基盤としてもよく使われるデータ構造です。何か特定の操作についてもっと詳しく知りたい場合は聞いてください！

確かに直接使う機会は少ないですが、いくつかシーンはあります。

---

## 直接使うケース

**undo/redo機能**
テキストエディタなどの操作履歴管理です。新しい操作を先頭に追加していくだけなので、片方向リストの「先頭追加がO(1)」という特性がマッチします。
```
最新 → [入力"a"] → [入力"b"] → [削除] → null
```

**ブラウザの戻る機能**
「前のページ」だけ持てればいいケースは片方向リストで十分です。

---

## 間接的に使っているケース（こちらの方が多い）

実務では直接実装するより、**内部でリスト構造を使っているライブラリやフレームワークを使う**ことがほとんどです。

例えばJavaだと…
- `LinkedList` クラス → 双方向リストだが同じ概念
- `HashMap` → 内部のハッシュ衝突の解決に片方向リストを使用

---

## 正直なところ

実務では配列（`ArrayList`）や`HashMap`で大体のケースはカバーできるため、**片方向リストをゼロから実装する機会はほぼありません。**

ただし…

| 目的 | 重要度 |
|------|--------|
| データ構造の概念理解 | ⭐⭐⭐ 高い |
| 参照・ポインタの理解 | ⭐⭐⭐ 高い |
| アルゴリズム面接対策 | ⭐⭐⭐ 高い |
| 実務での直接実装 | ⭐ 低い |

**「直接使うもの」というより「より高度なデータ構造を理解するための基礎」**として重要です。片方向リストが理解できると、木構造やグラフといった複雑なデータ構造の理解がスムーズになります。
